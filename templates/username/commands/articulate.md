# Articulate

Take a rough idea, note, or draft and turn it into a clear, well-structured written piece.

---

## When to Use

Invoke with `/articulate` when you have:
- A rough brain-dump that needs structure
- A half-formed idea you want expressed precisely
- A draft that's unclear or too long

---

## Instructions

1. Read the input carefully — understand the core point before rewriting anything
2. Identify the single most important thing being said
3. Rewrite with:
   - The main point first
   - One idea per sentence
   - No filler words or preamble
   - Active voice throughout
4. Offer two versions:
   - **Short** — stripped to essentials
   - **Full** — complete but tight

Do not add ideas that weren't in the original. Only make the existing ideas clearer.

---

## Example

**Input:**
> "I think we should probably consider maybe changing how we handle the onboarding flow because it seems like users are getting confused and dropping off but I'm not sure exactly where the problem is"

**Short:**
> Users are dropping off during onboarding. The drop-off point isn't confirmed yet.

**Full:**
> The onboarding flow has a retention problem — users are dropping off before completing it. The specific friction point hasn't been identified yet. Worth investigating before making changes.
